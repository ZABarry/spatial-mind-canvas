# Success criteria — thesis validation

Use these as **directional** signals, not rigid gates. Combine qualitative notes with the metrics in [templates/metrics.md](templates/metrics.md).

## Confirming evidence (examples)

- **Task success:** Most participants complete the scripted map task in VR without leaving for desktop-only actions.
- **Ease:** Median self-reported ease ≥ 4/5 after one guided session.
- **Structure recall:** After a distraction task, participants recall more node relationships from their VR map than from their 2D map (same person, counterbalanced order).
- **Intent:** ≥ 50% say they would use the spatial tool again for similar tasks.

## Falsifying evidence (examples)

- Participants consistently **prefer 2D** for the same task with lower time-on-task in 2D.
- **Motion discomfort** or tool confusion dominates feedback despite onboarding.
- **No difference** in recall or confidence between conditions.

## Decision rule (lightweight)

If **two of three** qualitative themes align (flow, confidence, recall) **and** no strong falsifying pattern appears, proceed to another build cycle focused on polish. Otherwise, tighten core UX before adding breadth.
